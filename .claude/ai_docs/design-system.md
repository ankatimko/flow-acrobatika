# FLOW Acrobatics — Дизайн-система лендинга

> Единый документ для дизайнера, копирайтера и разработчика.
> Источники: бренд-бук FLOW (Print_Logo_Tagline.pdf) + рабочий файл Figma `Flow__1_.fig`.
> Применять для лендинга по акробатике и любых производных страниц в этом бренде.

---

## 0. Базовые принципы

**Бренд FLOW звучит так:** премиально, легко, по-сестрински, без агрессивного маркетинга.
**Лендинг по акробатике звучит так:** уверенно, конкретно, с конверсионным фокусом, на «ты».

**Три правила, которые всегда выше эстетики:**

1. **Реальное фото > иллюстрация.** У FLOW есть реальные клиентки. Силуэт уместен только там, где фото невозможно.
2. **Конверсия > красивости.** Если декоративный элемент мешает увидеть кнопку — он лишний.
3. **Один блок — одна мысль.** Если на блоке три плашки и две кнопки — это два блока, разделить.

## 1. Цветовая система

### Primary (основные)

| Токен | HEX | Использование |
|---|---|---|
| `--gray-primary` | `#242427` | Тёмные фоны, основные заголовки на светлом |
| `--blue-primary` | `#0014FF` | Акценты, ссылки, левая часть фирменного градиента |
| `--pink-primary` | `#FF3CDF` | Главные CTA-кнопки, ключевые акценты, правая часть градиента |

### Семантические

| Токен | HEX | Использование |
|---|---|---|
| `--error` | `#FF2D55` | Ошибки валидации форм |
| `--warning` | `#FF9500` | Предупреждения |
| `--positive` | `#34C789` | Успешная отправка формы, галочки в чек-листах |

### Серые (нейтральные)

| Токен | HEX | Использование |
|---|---|---|
| `--gray-light` | `#6F707B` | Вторичный текст, мелкие подписи |
| `--gray-light-2` | `#9B9CA9` | Плейсхолдеры, разделители, отключенные элементы |
| `--white` | `#FFFFFF` | Текст на тёмных фонах, фоны блоков |

### Фирменный градиент

```css
background: linear-gradient(135deg, #0014FF 0%, #FF3CDF 100%);
```

Применяется на:
- Кнопки в hover-состоянии
- Декоративные blur-свечения на тёмных фонах
- Иконку «O» в логотипе
- Цифры в больших метриках (опционально)

**Где НЕ применять градиент:**
- Текст заголовков и подзаголовков (всегда сплошной цвет)
- Большие плоскости (становится «обоиным»)
- Иконки UI (галочки, стрелки)

### Правила контраста

- На `#242427` пишем белым `#FFFFFF` или `#9B9CA9` для вторичного текста
- На белом — `#242427` или `#6F707B` для вторичного
- Чистый чёрный `#000000` **не использовать** — только `#242427`

## 2. Типографика

### Шрифты

**Заголовки и UI:** **Kodchasan** (Google Fonts)
**Основной текст:** **Inter** (Google Fonts)

Kodchasan — фирменный шрифт логотипа, на нём держится визуальная идентичность. Inter добавляем для длинных текстов, потому что Kodchasan тяжеловат для абзацев.

### Шкала размеров

**Десктоп:**

| Уровень | Размер | Шрифт | Вес | Line-height | Letter-spacing |
|---|---|---|---|---|---|
| H1 | 64–72px | Kodchasan | Light/Regular | 1.05 | -0.02em |
| H2 | 40–48px | Kodchasan | Regular | 1.1 | -0.01em |
| H3 | 24–28px | Kodchasan | Medium | 1.2 | normal |
| H4 | 20px | Kodchasan | Medium | 1.3 | normal |
| Body Large | 18px | Inter | Regular | 1.5 | normal |
| Body | 16px | Inter | Regular | 1.5 | normal |
| Caption | 14px | Inter | Regular | 1.4 | normal |
| Small | 12px | Inter | Medium | 1.3 | 0.02em |

**Мобайл — уменьшаем заголовки:**

| Уровень | Размер |
|---|---|
| H1 | 36px |
| H2 | 28px |
| H3 | 20px |
| Body Large | 16px |
| Body | 16px |

### Регистр

- **H1, H2** — sentence case или UPPERCASE (на выбор для каждого блока, зависит от композиции)
- **Заголовки секций бренд-бука** — UPPERCASE с трекингом (как `MAIN LOGO`, `ADDITIONAL SIGNS`)
- **Кнопки** — UPPERCASE с трекингом 0.05em
- **Body** — обычный регистр

### Запрещено

- Цветной курсив отдельных слов в заголовках (приём 2010-х, удешевляет)
- Радужный или градиентный текст
- Больше двух шрифтов на странице
- Comic Sans, Times New Roman, системные шрифты

## 3. Сетка и отступы

### Контейнер

- **Десктоп:** 1280px max-width, центрирование, паддинги по 80px по бокам
- **Планшет (768–1024px):** паддинги 40px
- **Мобайл (<768px):** паддинги 20px

### Колоночная сетка

**Десктоп:** 12 колонок, gutter 24px
**Планшет:** 8 колонок, gutter 16px
**Мобайл:** 4 колонки, gutter 16px

### Вертикальные отступы между блоками

- Между крупными секциями: **120px** на десктопе, **64px** на мобайле
- Внутри блока, между заголовком и контентом: **48px** / **24px**
- Внутри карточки: **24px** / **16px**

### Шкала отступов (token system)

```
4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 80 / 120 / 160
```

Использовать только эти значения. Никаких 13px, 27px, 50px.

## 4. Скругления и тени

### Border radius

| Токен | Значение | Применение |
|---|---|---|
| `--radius-sm` | 8px | Кнопки, инпуты, мелкие чипы |
| `--radius-md` | 12px | Карточки FAQ, мелкие блоки |
| `--radius-lg` | 20px | Большие карточки (кейсы, тарифы) |
| `--radius-xl` | 32px | Крупные баннеры, изображения |
| `--radius-full` | 9999px | Pills, круглые кнопки |

### Тени

```css
--shadow-sm: 0 2px 8px rgba(36, 36, 39, 0.08);
--shadow-md: 0 8px 24px rgba(36, 36, 39, 0.12);
--shadow-lg: 0 16px 40px rgba(36, 36, 39, 0.16);
--shadow-pink: 0 8px 32px rgba(255, 60, 223, 0.24); /* для розовых CTA */
```

**На тёмном фоне** теней не используем — они не работают. Вместо тени — лёгкое внутреннее свечение или граница 1px `rgba(255,255,255,0.08)`.

## 5. Компоненты UI

### 5.1. Кнопки

#### Primary (главная CTA)

```
Фон: #FF3CDF
Текст: #FFFFFF
Шрифт: Kodchasan Medium, 16px, UPPERCASE
Высота: 56px
Padding: 16px 32px
Скругление: 12px
Hover: фон → linear-gradient(135deg, #0014FF, #FF3CDF) + тень --shadow-pink
Active: scale(0.98)
Disabled: opacity 0.4, cursor: not-allowed
```

**Используется:** ровно одна на экран. На hero — главная CTA. Дальше повторяется в финальном блоке формы.

#### Secondary (outline)

```
Фон: transparent
Текст: #FFFFFF (на тёмном) или #242427 (на светлом)
Обводка: 1px solid currentColor
Высота: 56px
Hover: фон → currentColor с opacity 0.08
```

**Используется:** в шапке («Записаться» в навигации), в карточках для второстепенных действий.

#### Tertiary (text link)

```
Фон: нет
Текст: #0014FF, подчёркивание при hover
```

**Используется:** «Узнать больше», «Подробнее», ссылки в FAQ.

#### Размеры

| Размер | Высота | Padding | Шрифт |
|---|---|---|---|
| Large | 56px | 16/32 | 16px |
| Medium | 48px | 12/24 | 14px |
| Small | 40px | 8/20 | 14px |

### 5.2. Формы

**Инпут:**
```
Высота: 56px
Padding: 0 20px
Фон (на светлом): #FFFFFF, обводка 1px #9B9CA9
Фон (на тёмном): rgba(255,255,255,0.06), обводка 1px rgba(255,255,255,0.12)
Скругление: 12px
Шрифт: Inter Regular, 16px
Focus: обводка 2px #0014FF
Error: обводка 2px #FF2D55, текст ошибки снизу #FF2D55
Placeholder: #9B9CA9
```

**Лейбл над инпутом:** Inter Medium, 14px, цвет `#6F707B`, отступ снизу 8px.

**Чекбокс согласия:** 20px квадрат, скругление 4px, активное состояние — заливка `#FF3CDF` с белой галочкой.

### 5.3. Карточки

**Базовая карточка:**
```
Фон: #FFFFFF (светлая) или rgba(255,255,255,0.04) (тёмная)
Скругление: 20px
Padding: 32px
Тень: --shadow-md (только на светлом фоне)
Hover: подъём -4px, тень --shadow-lg
```

**Карточка кейса:**
- Сверху — изображение/силуэт, 100% ширины
- Под ним — мета-строка, заголовок-цитата, текст
- Высота фиксированная для выравнивания в сетке

**Карточка тарифа:**
- Заголовок — название тарифа
- Цена крупно
- Список включений с галочками `--positive`
- Кнопка снизу
- Активный тариф (Хит / Максимум) — обводка `#FF3CDF` 2px + бейдж в углу

### 5.4. Бейджи и чипы

**Бейдж «Хит»:**
```
Фон: #FF3CDF
Текст: #FFFFFF
Шрифт: Kodchasan Medium, 12px, UPPERCASE
Padding: 6px 12px
Скругление: 9999px
```

**Чип категории (направления):**
```
Фон: rgba(0, 20, 255, 0.08)
Текст: #0014FF
Padding: 8px 16px
Скругление: 9999px
```

### 5.5. Иконки

**Стиль:** outline, 1.5px stroke, скругление концов.
**Размеры:** 16 / 20 / 24 / 32px
**Источник:** Lucide icons (open source, бесплатные)

**Запрещено:** разноцветные плоские иконки в стиле Material, эмодзи в основном UI (только в подзаголовках секций для эмоционального акцента).

### 5.6. Навигация

**Шапка десктоп (height 80px):**
- Логотип FLOW слева (40px высоты)
- Меню по центру: ЗАЧЕМ · КЕЙСЫ · УРОВНИ · ТРЕНЕРЫ · СТУДИИ · ЦЕНЫ · FAQ (Kodchasan Medium, 14px, UPPERCASE)
- Телефон + кнопка «Записаться» (outline, не filled) справа

**На скролле:**
Шапка становится sticky, фон `#242427` с blur backdrop, высота уменьшается до 64px.

**Шапка мобайл (height 56px):**
- Логотип
- Бургер-меню
- Иконка «Записаться» (телефон или щепотка) — открывает экспресс-форму

**Правила использования фото:**

3. Не превращать в силуэт через filter: brightness(0).
4. Не обрезать так, чтобы потерялась голова или ноги (это акробатика — важна вся фигура).

### 6.2. Контурные иллюстрации (приоритет 2)

**Что есть:**
- Розовый неоновый контур позы голубя
- Чёрный контур позы балерины
- Чёрный контур мостика на руках

**Где применять:**
- Декоративные акценты в углах блоков
- Иконки направлений (BASE, MIDDLE, JUMP)
- Фоновые элементы с пониженной opacity (15–25%)

**Где НЕ применять:**
- Hero (там нужно живое фото)
- Кейсы клиенток (там нужны конкретные результаты, а не абстракции)

### 6.3. Декоративные элементы (приоритет 3)

**Фирменные «волны» / полумесяцы из логотипа:**
- 5 вариантов в файле, разные позы и цвета (бирюзовые, голубые)
- Применять как фоновую графику, opacity 30–50%, blur 0–4px
- Размер: 200–600px по большей стороне
- **Не более одной волны на блок.** Перегружает.

**Синие blob-пятна:**
- Применять как мягкий фоновый акцент за карточками
- Opacity 60–80%, blur 40–80px
- Не использовать как кнопки или контейнеры

**Маленькие синие капли/точки:**
- Декоративные акценты в местах списков
- Размер 16–32px
- Можно использовать как маркеры буллетов

### 6.4. Логотип

**Версии:**
- Основной: `FLOW` с фирменным «O»-волной (на светлом — чёрный текст, на тёмном — белый)
- С тэглайном: `FLOW · FEM TRAINING LAB` — для подвалов и печати
- Только знак: «O»-волна — для favicon и аватарок

**Минимальные размеры:**
- Десктоп: 120px по ширине
- Мобайл: 80px
- Favicon: 32px (только знак)

**Охранное поле:** минимум высота буквы «F» вокруг логотипа.

**Запрещено:**
- Растягивать непропорционально
- Менять цвет «O»-волны
- Накладывать на пёстрый фон без подложки
- Поворачивать
- Дублировать

### 6.5. Прочие ассеты

**Mockup телефона в руке** — для блока про мобильное приложение FLOW.
**Фасад студии** — для блока «Студии».
**Интерьер с розовой подсветкой** — для блока «Атмосфера» (если будет).
**QR-коды** — генерировать заново под актуальные ссылки, существующие в файле могут быть устаревшие.

## 8. Анимация

### 8.1. Базовые принципы

- **Только то, что усиливает контент.** Не двигаем «потому что красиво».
- **Длительность:** 200–400ms для микро-взаимодействий, 600–800ms для появления блоков.
- **Easing:** `cubic-bezier(0.4, 0, 0.2, 1)` (ease-out) по умолчанию.

### 8.2. Допустимые анимации

| Где | Что | Длительность |
|---|---|---|
| Кнопки | hover: scale(1.02), смена фона | 200ms |
| Карточки | hover: подъём -4px | 300ms |
| Появление блоков на скролле | fade-in + slide-up 24px | 600ms |
| Свечения на фоне hero | очень медленный сдвиг | 8000ms loop |
| Изображение в hero | float ±4px по вертикали | 6000ms loop |
| Open/close FAQ | высота + opacity | 300ms |

### 8.3. Запрещённые анимации

- Параллакс-скролл с большим количеством слоёв
- Анимация по курсору (особенно «магнитные кнопки»)
- Печатающийся текст в заголовках
- 3D-флипы карточек
- Long-press интерактивы на десктопе
- Любая автоматическая карусель быстрее 5 секунд

## 9. Адаптивность

### 9.1. Брейкпоинты

```
Mobile:    < 768px
Tablet:    768px – 1023px
Desktop:   1024px – 1439px
Large:     ≥ 1440px
```

### 9.2. Mobile-first

- Макеты сначала рисуем под мобайл, потом раскатываем
- 70%+ трафика будет с телефонов — мобайл это default, не «адаптация после»
- На мобайле все важные действия — в зоне большого пальца (нижняя половина экрана)

### 9.3. Touch targets

Минимум 48×48px для всех интерактивных элементов на мобиле.
Расстояние между кликабельными элементами — минимум 8px.

### 9.4. Sticky CTA на мобайле

Внизу экрана всегда висит кнопка «Записаться на пробное» — высотой 56px, с фирменным розовым фоном.
Скрывается только когда пользователь долистал до финальной формы.

---

## 10. Доступность

- **Контраст текста:** минимум 4.5:1 (WCAG AA), для крупного текста 3:1.
- **Альт-тексты** на всех изображениях (особенно у фото клиенток для SEO).
- **Aria-labels** на иконочных кнопках и формах.
- **Focus-visible** на всех интерактивных элементах: обводка 2px `#0014FF` с offset 2px.
- **Keyboard navigation** — все CTA доступны с клавиатуры.
- **Уменьшенная анимация:** при `prefers-reduced-motion: reduce` отключаем все loop-анимации, оставляем только мгновенные переходы.

## 11. Производительность

### 11.1. Изображения

- Все фото: формат **WebP** с fallback в JPEG для старых браузеров
- Hero-фото: предзагружается (`<link rel="preload">`)
- Остальные фото: lazy-loading через `loading="lazy"`
- Размеры: srcset под 1x / 2x для retina
- Никаких PNG для фото с фоном — только для прозрачных силуэтов и иконок

### 11.2. Шрифты

- Подключаем через `<link rel="preload">` с `font-display: swap`
- Только нужные начертания (Kodchasan Light + Regular + Medium, Inter Regular + Medium)
- Subset для кириллицы и латиницы

### 11.3. Таргеты PageSpeed

- LCP: < 2.5s
- FID: < 100ms

## 12. Голос и подача (cross-reference)

Подробный tone of voice вынесен в отдельный документ «Бренд-гайд».
Здесь — краткие напоминалки для UI-копирайтинга:

- **Обращение** — на «ты»
- **Кнопки** — глагол + конкретика: «Записаться на пробное», не «Подробнее»
- **Микро-копирайтинг форм** — без формальностей: «Имя» вместо «Ф.И.О.», «Телефон» вместо «Контактный номер»
- **Подтверждения** — тёплые: «Спасибо! Перезвоним за 15 минут», не «Ваша заявка принята»
- **Ошибки** — без обвинений: «Кажется, телефон не дописался», не «Введён некорректный номер»
